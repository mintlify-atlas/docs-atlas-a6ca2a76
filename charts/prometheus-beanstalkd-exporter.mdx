title: Prometheus Beanstalkd Exporter
description: Export Beanstalkd queue metrics to Prometheus
---

The Prometheus Beanstalkd Exporter chart deploys a Prometheus exporter for [Beanstalkd](https://beanstalkd.github.io/) work queues, exposing metrics about queue tubes, jobs, and server statistics.

## What It Monitors

This exporter collects metrics from Beanstalkd servers including:

- Tube statistics (ready, buried, delayed, urgent jobs)
- Job counts and states
- Server connections and commands
- Queue depth and processing metrics

## Installation

### Basic Installation

```bash
helm repo add douban https://douban.github.io/charts
helm install beanstalkd-exporter douban/prometheus-beanstalkd-exporter \
  --set beanstalkd.address="mybeanstalkd:11300"
```

## ServiceMonitor Configuration

To enable automatic Prometheus scraping via the Prometheus Operator:

```yaml
serviceMonitor:
  enabled: true
  interval: 30s
  labels:
    prometheus: kube-prometheus
```

<ParamField path="serviceMonitor.enabled" type="boolean" default="false">
  Enable ServiceMonitor resource creation
</ParamField>

<ParamField path="serviceMonitor.interval" type="string" default="30s">
  How frequently Prometheus should scrape
</ParamField>

<ParamField path="serviceMonitor.timeout" type="string" default="10s">
  Scrape timeout duration
</ParamField>

## Manual Prometheus Configuration

If not using the Prometheus Operator, add this scrape config:

```yaml
scrape_configs:
  - job_name: 'beanstalkd-exporter'
    static_configs:
      - targets: ['beanstalkd-exporter:8080']
    scrape_interval: 30s
    scrape_timeout: 10s
```

## Prometheus Rules

You can define custom alerting rules:

```yaml
prometheusRule:
  enabled: true
  rules:
    - alert: BeanstalkdTubeBacklog
      expr: beanstalkd_current_jobs_ready > 1000
      for: 5m
      labels:
        severity: warning
      annotations:
        summary: Beanstalkd tube has high backlog
        description: "Tube {{ $labels.tube }} has {{ $value }} ready jobs"
```

## Metrics Exposed

The exporter provides metrics at the `/metrics` endpoint including:

- `beanstalkd_current_jobs_ready` - Jobs ready to be processed
- `beanstalkd_current_jobs_reserved` - Jobs currently being processed
- `beanstalkd_current_jobs_delayed` - Jobs waiting to become ready
- `beanstalkd_current_jobs_buried` - Failed jobs that need manual intervention
- `beanstalkd_cmd_put_total` - Total put commands
- `beanstalkd_current_connections` - Current client connections
